# Grok Split Sync (v2.0)

A skill designed to slice massive Grok/xAI database JSON exports (such as `prod-grok-backend.json`) into chronologically partitioned archives. Conversations are saved as individual JSON objects inside month-level, week-level, and day-level subdirectories.

Version 2.0 introduces a highly robust interactive command-line interface (Terminal GTUI) with a stepping wizard, help menus, rabbits and snakes, error logging, and procedural logs written to disk.

## When to Use

- When processing extremely large conversation databases exported from xAI/Grok.
- When needing a day-by-day temporal reconstruction where conversations are duplicated in the folders of every single calendar date on which they had recorded activity (were "touched").
- When generating clean, sanitized, and filesystem-safe filename structures for Obsidian knowledge management ingestion.
- When requiring detailed progress tracking and error debugging with logs saved to disk.

## Features

- **Interactive Wizard Mode**: Automatically boots into a friendly step-by-step terminal UI if run without arguments or with `--interactive` / `-i`.
- **Stepping Engine**: Supports a stepping process (`next`, `next`, `all`, `abort`) where you can audit each conversation's details, process it, skip it, process all remaining files, or safely abort the operation.
- **Log Management**:
  - **Procedural Logs**: Detailed records of file validations, parsed dates, subfolder structures, and successfully saved archives saved to `<output_dir>/logs/procedural/process_log_YYYYMMDD_HHMMSS.txt`.
  - **Error Logs**: Clean records of parsing anomalies, data mismatches, and execution tracebacks saved to `<output_dir>/logs/errors/error_log_YYYYMMDD_HHMMSS.txt`.
- **Backward Compatibility**: Supports immediate command-line execution for background server tasks when input and output arguments are supplied.

## Steps

1. **Launch the Splitter**:
   Run the utility on your VM in your preferred mode:
   
   - **Interactive Mode (Recommended)**:
     ```bash
     python3 scripts/grok_splitter.py
     ```
   
   - **Non-Interactive Mode**:
     ```bash
     python3 scripts/grok_splitter.py <input_json_path> <output_root_directory>
     ```

2. **Walk Through the Wizard (Interactive)**:
   - **Step 1**: Read the welcome banner (with bunny and snek) and press `next`.
   - **Step 2**: Select from detected JSON files or input a path. Input structure will be validated.
   - **Step 3**: Specify or accept the default output den. Write access and log systems will be verified and initialized on disk.
   - **Step 4**: Choose `run` to digest everything, or `step` to manually audit and step through conversations one-by-one.

3. **Verify structured folder layout**:
   - Confirm that output folders are organized as `grok_archive/YYYY_MM_MonthName/Week_WW/YYYY-MM-DD/`.
   - Review `logs/procedural/` and `logs/errors/` for processing milestones and error logs.

## Gotchas

- **File size limits**: When running on massive files (e.g., >500MB), ensure the sandbox VM has adequate disk space allocated before executing.
- **MIME Type Blocks on Drive**: Renaming file extensions to bypass Google Drive's REST API blocks does not change their internal MIME types. Processing is most reliably executed on local files inside this VM environment.
- **Asynchronous uploads**: If GDrive desktop client is active, it may take time to propagate the hundreds of micro-files generated by the splitter. Check the logs systematically.
